# AGENTS.md - TIL 운영 규칙

이 문서는 TIL 저장소의 **전역 운영 규칙(Single Source of Truth)** 이다.

---

## 1) 언어 정책
- **모든 문서는 한글로 작성**한다.
- 기술 용어/고유명사는 필요 시 **원문(영문)을 병기**한다.

## 2) 팩트/출처 정책 (Authoritative Sources)
- 원칙: **공식 문서(Documentation) / 공식 표준 / 공식 제안서(KIP/KEP/PEP 등) / 공식 GitHub(프로젝트 본체)** 를 최우선으로 신뢰한다.
- **최소 3개 이상의 권위 소스**로 교차 검증한다. (가능하면 “서로 독립된” 출처)
- 블로그/세컨더리 자료는 참고 가능하나, 최종 결론은 반드시 공식 근거로 수렴시킨다.

## 3) CONTEXT.md 운영 규칙
- 작업 시작 시 해당 디렉터리의 `CONTEXT.md`를 **반드시 읽고** 규칙/목표/성숙도 상태를 확인한다.
- 작업 완료 시 `CONTEXT.md`의 아래 항목을 **반드시 최신화**한다.
  - Authoritative Sources(공식 출처) 구체화
  - Technical Maturity(학습 성숙도) 체크
  - 폴더 전용 규칙(있다면) 준수 여부

## 4) PRD(계획 문서) 프로세스 (PRD-First)
- 복잡한 작업(문서 구조 변경, 범위가 큰 신규 문서, 리팩터링/재작성)은 **PRD-First**로 진행한다.
  - Planning → Agreement → Implementation
- **PRD는 내부 계획/범위 정리용 문서**로 취급한다.
- 원칙: **PR/리포지토리에 PRD 파일을 포함하지 않는다.**
  - 대신 PR Description(설명) 또는 커밋 메시지에 범위/의도를 요약한다.

## 5) PR/브랜치 운영 규칙
- 동일 주제에 PR이 이미 열려 있으면 **새 PR을 만들지 않고 기존 PR을 업데이트**한다.
- Fix/Refine 성격의 변경은 기존 PR에 누적하여 리뷰 연속성을 유지한다.
