# 쿠버네티스란? (What is Kubernetes?)

## 개념 (Concept)
- Go로 작성된 오픈 소스 컨테이너 오케스트레이션 플랫폼
- 컨테이너화된 애플리케이션의 배포(Deployment), 확장(Scaling), 관리(Management) 자동화
- Google의 내부 시스템 Borg에서 파생된 프로젝트로 CNCF(Cloud Native Computing Foundation)에서 관리

## 역할 (Purpose)
- 선언적 API(Declarative API)를 통한 의도(Intent) 기반 관리
- 다중 노드(Multi-node) 환경에서 컨테이너 스케줄링 및 로드 밸런싱
- Self-healing, Rolling update, Rollback 등 자동화된 운영 기능 제공
- 하이브리드/멀티 클라우드 환경 지원

## 주요 기능 (Key Features)
- **자동화된 스케줄링 (Automated Scheduling)**: 리소스 요구사항과 제약 조건 기반으로 노드에 컨테이너 배치
- **셀프 힐링 (Self-healing)**: 장애 발생 시 자동으로 컨테이너 재시작, 재스케줄링, 노드 교체
- **롤링 업데이트/롤백 (Rolling Update & Rollback)**: 서비스 중단 없는 배포 및 이전 버전으로의 빠른 복구
- **서비스 디스커버리 (Service Discovery)**: DNS 또는 환경 변수를 통한 서비스 검색
- **로드 밸런싱 (Load Balancing)**: Service를 통한 트래픽 분산

## 작동 원리 (How It Works)
- **선언적 관리**: 사용자는 원하는 상태(Desired State)만 정의 (예: "3개의 복제본 실행")
- **Control Plane**: 현재 상태(Actual State)를 모니터링하고 원하는 상태와 일치하도록 조정
- **Reconciliation Loop**: 현재 상태와 원하는 상태의 차이를 인지하고 실제 상태를 원하는 상태로 수렴

## 장애 처리 (Failure Handling) ⭐중요
- **노드 장애 (Node Failure)**
  - 동작 방식: Kubelet 하트비트 손실로 Node NotReady 상태 감지
  - 복구 메커니즘: 해당 노드의 Pod를 다른 건강한 노드로 재스케줄링 (Reschedule)

- **컨트롤 플레인 장애 (Control Plane Failure)**
  - 동작 방식: API Server 장애 시 클러스터 관리 불가 (기존 작동은 계속)
  - 복구 메커니즘: 다중 Master 구성(High Availability)으로 단일 장애점(SPOF) 제거

- **etcd 장애 (etcd Failure)**
  - 동작 방식: 데이터 일관성 보장을 위해 읽기/쓰기 불가
  - 복구 메커니즘: Raft 합의 알고리즘을 통한 리더 선출 및 복구, 정기적 백업 필수

## 사용 사례 (Use Cases)
- 마이크로서비스 아키텍처 (Microservices Architecture) 배포
- CI/CD 파이프라인 통합 및 자동화
- 대규모 스테이트리스(Stateless) 애플리케이션 운영
- 배치 작업(Batch Job) 및 일정 관리 (CronJob)
